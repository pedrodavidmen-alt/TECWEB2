El servidor procesa la solicitud

El servidor web (como Apache, Nginx o Node.js) recibe la petición, interpreta el método HTTP y busca el archivo solicitado en su almacenamiento o consulta una base de datos si el contenido es dinámico.

Ejemplo de encabezados de respuesta HTTP

Tras procesar la solicitud, el servidor genera una respuesta acompañada de metadatos indispensables:

Campos comunes en una respuesta HTTP
Línea / encabezado Valor de ejemplo Significado
Código de estado 200 OK Informa que la solicitud fue exitosa y el recurso existe.
Content-Type text/html; charset=UTF-8 Informa al navegador que el archivo enviado es un documento HTML.
Código de estado 404 Not Found El recurso pedido no fue encontrado en el servidor.
Código de estado 500 Internal Server Error El servidor encontró un error interno al procesar la solicitud.
Pasa el cursor (o el foco del teclado) sobre cada código de estado para ver su significado.
