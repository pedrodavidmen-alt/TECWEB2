El navegador interpreta el HTML

El servidor envía el cuerpo de la respuesta con el código HTML puro. El navegador lee el archivo línea por línea y construye el DOM (Document Object Model) para pintar la interfaz visual en tu pantalla.

Importancia de la semántica en HTML5

Las etiquetas semánticas ayudan a los motores de búsqueda y a las tecnologías asistivas (como los lectores de pantalla) a entender la estructura del sitio. Estos son los elementos clave utilizados en este proyecto:

  • <header> Define la cabecera de la página o de una sección.
    Estructura
  • <nav> Contiene los bloques principales de enlaces de navegación.
    Navegación
  • <main> Define el contenido principal y único de todo el documento.
    Contenido
  • <section> Representa secciones genéricas con temáticas agrupadas.
    Contenido

Estructura del código recibido

El código básico que el navegador interpreta luce exactamente de la siguiente manera:

<!DOCTYPE html>
<html lang="es">
  <head>
    <title>Título de la Página</title>
  </head>
  <body>
    <main>Contenido principal aquí</main>
  </body>
</html>

Los lectores de pantalla recorren el DOM en el mismo orden en que aparece en el HTML. Por eso el orden lógico de las etiquetas semánticas (encabezados, secciones, listas) determina si la experiencia de navegación por teclado o por voz será clara o confusa.
