El navegador realiza una petición HTTP

Una vez localizada la dirección IP, el navegador (cliente) abre una conexión y envía un paquete de datos llamado Petición HTTP. Esta petición le comunica al servidor exactamente qué recurso quiere obtener.

Métodos HTTP principales

Comparativa de métodos HTTP
Método Descripción Cuándo se usa
GET Solicita datos de un recurso específico en el servidor. Al cargar una página web, una imagen o buscar información.
POST Envía datos al servidor para crear o actualizar un recurso. Al enviar un formulario de registro o subir un archivo.
Idempotencia: repetir una petición GET no cambia el estado del servidor; repetir una POST puede crear registros duplicados.

Otros métodos HTTP (para profundizar)

Reemplaza por completo el recurso indicado en la URL con los datos enviados.

Solicita al servidor que elimine el recurso identificado por la URL.

Modifica solo algunos campos del recurso, sin reemplazarlo por completo.

Para aprender más sobre cómo estructurar estas peticiones, puedes visitar la documentación oficial de MDN sobre HTTP (se abre en una pestaña nueva) .

Cliente y servidor

Cliente: el programa que inicia la solicitud, normalmente tu navegador.

Servidor: la computadora remota que almacena los recursos (páginas, imágenes, datos) y responde a las peticiones del cliente.
